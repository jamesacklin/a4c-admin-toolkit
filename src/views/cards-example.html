Card Examples

These are some example uses for cards with the different data types that exist in the admin app today. You'll notice that each card uses a different combination of content elements—mostly combinations of raw paragraphs, data grids, and tables. Some cards, like the Event card, have advanced functionality and interactivity built into them.

Events

Events have an event title (housed in the card title), an event description (contained in a <blockquote>), and a data grid showing some meta elements.

Events also contain a collapsing table showing the various RSVP statuses of club members who have responded. The user can change club members' RSVP statuses by dragging and dropping between columns. Planned: on small screens, the user can change club members' statuses by tapping on a member's name and selecting a new RSVP status via a dropdown menu.

If the Payments module is enabled, arrange the data grid with 4 columns at medium and large breakpoints (.grid-md-4, .grid-lg-4).
