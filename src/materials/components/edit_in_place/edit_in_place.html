--- notes: | To provide users a way to quickly edit values on a without editing the entire data construct (flipping the Beverage Cart status from Not Running to Running, for example), make liberal use of inline editing. We make use of the X-editable library for this. (_This is a lie but will soon be true: the library is included in `toolkit.js` and the requisite styles are included in `toolkit.css`. The bundled version of X-editable works with Bootstrap 3._) Your best bet is to follow the X-editable docs for exact implementation instructions, but in essence, add all options inline to the link that you want to enact edit-in-place functionality upon. The toolkit JavaScript picks up on the `data-editable` attribute, and X-editable handles the rest. From the X-editable docs, here are the main attributes you should define: 
    type - type of input (text, textarea, select, etc)
    url - url to server-side script to process submitted value (/post, post.php etc)
    pk - primary key of record to be updated (ID in db)
    id or name - name of field to be updated (column in db). Taken from id or data-name attribute
    value - initial value. Usefull for select, where value is integer key of text to be shown. If empty - will be taken from element html contents
Check out the X-editable demos for more (the select and checkbox `data-type`s should be of particular interest). ---
